Code Documentation Application

Analyze codebases and generate comprehensive documentation using LLM technology

Application Overview

This application analyzes GitHub repositories or zip files containing codebases and generates comprehensive documentation using LLM technology. It includes a RAG system for context-aware code understanding and can generate Q&A pairs for development and feature changes.

Key Features

  • Process code from GitHub repositories or zip files
  • Analyze code structure and dependencies
  • Generate comprehensive documentation with LLM assistance
  • Create 50 Q&A items about codebase development and feature changes
  • Provide context-aware code understanding through RAG

API Endpoints

Health Check

GET /

Check if the API is running

Response Example:
{
  "status": "healthy",
  "message": "Code Documentation API is running"
}

Documentation Generation

POST /api/github

Process GitHub repository and generate documentation

Request Example:
{
  "repo_url": "https://github.com/username/repo",
  "output_dir": "optional/output/path",
  "llm_config": {
    "base_url": "https://api.example.com",
    "api_key": "your_api_key",
    "instruct_model": "model_name",
    "reasoning_model": "model_name",
    "embeddings_model": "model_name"
  }
}
Response Example:
{
  "task_id": "123e4567-e89b-12d3-a456-426614174000",
  "status": "processing",
  "output_dir": "/path/to/output",
  "message": "GitHub repository processing started"
}
POST /api/zip

Process zip file and generate documentation

Request Example:
{
  "file_path": "/path/to/codebase.zip",
  "output_dir": "optional/output/path",
  "llm_config": {
    "base_url": "https://api.example.com",
    "api_key": "your_api_key",
    "instruct_model": "model_name",
    "reasoning_model": "model_name",
    "embeddings_model": "model_name"
  }
}
Response Example:
{
  "task_id": "123e4567-e89b-12d3-a456-426614174000",
  "status": "processing",
  "output_dir": "/path/to/output",
  "message": "Zip file processing started"
}

Task Management

GET /api/tasks/{task_id}

Get task status

Response Example:
{
  "task_id": "123e4567-e89b-12d3-a456-426614174000",
  "status": "completed",
  "progress": 100,
  "message": "Documentation generation completed",
  "result": {
    "file_count": 10,
    "function_count": 20,
    "class_count": 5,
    "output_path": "/path/to/output"
  }
}

RAG Queries

POST /api/query

Query documentation using RAG

Request Example:
{
  "query": "How does the authentication system work?",
  "task_id": "123e4567-e89b-12d3-a456-426614174000",
  "max_results": 5
}
Response Example:
{
  "query": "How does the authentication system work?",
  "answer": "The authentication system uses JWT tokens...",
  "context": [
    {
      "content": "def authenticate_user(username, password): ...",
      "metadata": {"file_path": "/path/to/auth.py"},
      "distance": 0.1
    }
  ]
}

Q&A Retrieval

GET /api/qa/{task_id}

Get Q&A pairs for a task

Response Example:
{
  "task_id": "123e4567-e89b-12d3-a456-426614174000",
  "qa_pairs": [
    {
      "question": "How do I add a new feature to the application?",
      "answer": "To add a new feature, you would need to..."
    },
    {
      "question": "What is the architecture of the codebase?",
      "answer": "The codebase follows a modular architecture with..."
    }
  ]
}

Environment Variables

Variable Description Default
LLM_BASE_URL Base URL for LLM API https://api.example.com
LLM_API_KEY API key for LLM service
LLM_INSTRUCT_MODEL Model name for instruction following instruct_model
LLM_REASONING_MODEL Model name for reasoning tasks reasoning_model
LLM_EMBEDDINGS_MODEL Model name for embeddings generation embeddings_model
OUTPUT_DIR Output directory for documentation documentation_output
TEMP_DIR Temporary directory for processing /tmp/code_documentation
LOG_LEVEL Logging level INFO
API_HOST Host to bind the API server 0.0.0.0
API_PORT Port to bind the API server 8000
API_WORKERS Number of worker processes 4
API_DEBUG Enable debug mode false
API_CORS_ORIGINS Allowed CORS origins *
API_RATE_LIMIT Rate limit for API requests 100
VECTOR_DB_PATH Path to vector database for RAG vector_db
